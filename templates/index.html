Wayback Machine Submitter (testing)

Submits each URL to the Wayback Machine's Save Page Now to create a fresh, dated snapshot - useful as a trusted citation/backup independent of the SEO reports. Rotates through your proxy pool since archive.org blocks by IP; each URL gets up to 3 tries.

Max 20 URLs per batch - archive.org rate-limits/blocks bulk submissions. Run additional batches for more. Pure HTTP - no browser needed.
SEranking Audit (testing)

Upload a SEranking Site Audit .xlsx export (or your own similar issue-list export) and get back the same sheets with suggestion columns filled in - real page-specific Title/H1/Description suggestions and broken-link fixes, generated from each page's live content. Raw technical sheets (JS/CSS/image issues) pass through unchanged. Suggestions only - nothing is ever changed on your live site.

Can take a few minutes - each flagged page's live content is fetched to generate real suggestions. PDF and legacy .xls (zip) reading use a proper parsing library, installed automatically into this app's own Python the first time you use it (needs internet access once); if that install can't complete, PDF falls back to a best-effort text extraction and reports honestly when it can't reliably read a page.
GEO / AI Optimization (testing)

Generates the full monthly GEO deliverable set from just a domain (+ pages/keywords if you have them): a Keywords/FAQ xlsx, llms-full.txt, Schema Suggestions (FAQPage JSON-LD), a Technical Optimization (ALT-tag) report, Content Modification suggestions, and Internal Linking suggestions (checked against the real sitemap, not just the pages in this run). The Keywords xlsx also checks - without login, without any paid API - whether the site is already cited in real AI-answer results for each query. Perplexity's check is reliable; Google AI Overview and ChatGPT are best-effort only (both commonly report "Not checked" - Google blocks headless requests with a captcha, ChatGPT requires a logged-in account for a real answer). Not yet included: the Performance Report (rank tracking / Ahrefs / GA4 screenshots) - separate, bigger scope.

Can take several minutes - generates 6 files (xlsx, llms-full.txt, and 4 docx), each with its own AI calls and, for the xlsx, one browser query per generated FAQ if visibility checking is on. All suggestions are verified against the page's real crawled content before being included - a suggestion whose text doesn't actually exist on the page is dropped, not guessed.
Performance Report (testing)

A standalone monthly performance deck (pptx) built from real Google Search Console + Google Analytics 4 data. Connect a GSC account below (the same accounts as GSC Audit); if it was connected before this feature existed, reconnect it once to grant the extra Analytics permission. GA4 requires the connected Google account to have Viewer access in that GA4 property. If that account also has a browser session (GSC Audit tab -> GSC Browser Sessions -> "+ New Session", logged in as the same account), the report includes real dashboard screenshots for whichever metrics (clicks/impressions/position, users/sessions) actually improved vs. the previous period - otherwise it falls back to native charts built from the same data.

This account needs to be reconnected to grant Analytics access.
This is "james" format - the first Performance Report layout. More formats will be added the same way On-Page formats were, once real client references exist for them. Rank-tracking (SE Ranking) and GSC Security/Links sections aren't included yet - separate data sources not wired up.
